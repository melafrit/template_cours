    Accessibilité & diffusionAccessibility & delivery

    Un cours n'existe vraiment que lorsqu'il atteint tout le monde, partout. Ce chapitre rend votre cours accessible, bilingue, utilisable hors-ligne, puis le publie sous licence ouverte. A course only truly exists once it reaches everyone, everywhere. This chapter makes your course accessible, bilingual, usable offline, then publishes it under an open licence.

    Pourquoi ce chapitre compteWhy this chapter matters

    Le meilleur contenu reste inutile s'il est illisible, inatteignable ou réservé à une langue. Soigner l'accessibilité et la diffusion, c'est respecter chaque apprenant et démultiplier la portée de votre travail. The best content stays useless if it is unreadable, unreachable or locked to one language. Caring about accessibility and delivery means respecting every learner and multiplying the reach of your work.

    1. Les bases de l'accessibilité1. Accessibility basics

    L'accessibilité n'est pas une option de fin de projet : elle se conçoit dès le départ. Quatre leviers simples couvrent l'essentiel : un contraste suffisant entre texte et fond, une navigation au clavier complète (sans souris), une police lisible à taille confortable, et le respect du mouvement réduit pour les personnes sensibles aux animations. Accessibility is not an end-of-project add-on: it is designed in from the start. Four simple levers cover the essentials: enough contrast between text and background, full keyboard navigation (no mouse), a readable font at a comfortable size, and respect for reduced motion for people sensitive to animations.

    📌

    Testez chaque page sans souris : si la touche Tab permet d'atteindre tous les liens et boutons dans un ordre logique, l'accessibilité clavier est gagnée.Test each page without a mouse: if the Tab key reaches every link and button in a logical order, keyboard accessibility is won.

    2. Concevoir un contenu bilingue2. Designing bilingual content

    Un cours bilingue n'est pas une traduction posée à côté : c'est deux versions équivalentes qui disent la même chose avec la même précision. Chaque bloc porte ses deux langues, et un seul interrupteur bascule l'ensemble. Cette discipline force la clarté : une idée floue se traduit mal, donc se repère vite. Pensez aussi à déclarer la langue dans le code (lang) pour que les lecteurs d'écran prononcent correctement. A bilingual course is not a translation pasted alongside: it is two equivalent versions saying the same thing with the same precision. Each block carries its two languages, and a single switch flips everything. This discipline forces clarity: a fuzzy idea translates badly, so it surfaces fast. Also declare the language in the code (lang) so screen readers pronounce it correctly.

    3. Fonctionner hors-ligne3. Working offline

    Un cours qui dépend d'un serveur disparaît dès que la connexion tombe. En gardant un site statique — HTML, CSS et JavaScript sans base de données — le cours s'ouvre depuis un simple dossier, une clé USB ou un cache local. Les flashcards, le QCM et la progression fonctionnent alors entièrement côté navigateur. C'est idéal pour le micro-apprentissage dans le train, en classe sans wifi ou sur un poste isolé. A course that depends on a server vanishes the moment the connection drops. By keeping a static site — HTML, CSS and JavaScript with no database — the course opens from a plain folder, a USB stick or a local cache. The flashcards, the quiz and the progress then run entirely in the browser. This is ideal for microlearning on a train, in a wifi-less classroom or on an isolated machine.

    💡

    Stocker la progression dans le navigateur (et non sur un serveur) garde le cours hors-ligne et respecte la vie privée : aucune donnée ne quitte l'appareil.Storing progress in the browser (not on a server) keeps the course offline and privacy-friendly: no data ever leaves the device.

    4. Publier, diffuser, partager4. Publishing, delivering, sharing

    Un site statique se publie sans serveur applicatif : GitHub Pages, ou tout hébergement de fichiers, suffit à le mettre en ligne gratuitement. Reste à choisir une licence ouverte : MIT pour le code, Creative Commons (par exemple CC BY 4.0) pour le contenu pédagogique. La licence dit clairement ce que d'autres peuvent réutiliser, adapter et partager — et à quelles conditions. Sans licence, le cours est, par défaut, « tous droits réservés ». A static site publishes without an application server: GitHub Pages, or any file hosting, is enough to put it online for free. The remaining choice is an open licence: MIT for the code, Creative Commons (for example CC BY 4.0) for the teaching content. The licence states clearly what others may reuse, adapt and share — and under which conditions. Without a licence, the course is, by default, "all rights reserved".

    À retenirKey takeaways

    • L'accessibilité se conçoit dès le départ, pas à la fin.Accessibility is designed in from the start, not bolted on at the end.
    • Un site statique fonctionne hors-ligne et se publie partout.A static site works offline and publishes anywhere.
    • Une licence ouverte autorise explicitement la réutilisation.An open licence explicitly authorises reuse.
